# Proof Structures

Use this reference when you need a stable structure for case studies or customer stories.

## Formal Case Study

Recommended flow:

1. Customer context
2. Problem
3. Solution
4. Implementation or change
5. Result
6. Quote if approved

## Customer Story

Recommended flow:

1. Starting point
2. Challenge
3. Turning point
4. Change
5. Outcome

## Sales Proof Summary

Recommended flow:

1. Who the customer was
2. What problem they faced
3. What changed
4. What result mattered most
